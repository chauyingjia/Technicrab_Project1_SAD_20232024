## 🏦 Reflection: Koperasi KADA Online System Project

Through this project, I gained a clearer understanding of how a real software system is planned, designed, developed, and tested. The Koperasi KADA Online System helped me understand how software can solve real business problems, especially in replacing manual paper-based processes with a more structured digital system.

### 💡 What I Gained

I learned how important requirement analysis and system design are before starting system development. This project helped me improve my understanding of **user stories**, **system scope**, **UML diagrams**, **database design**, and **user interface planning**. I also learned that a system should not only function correctly, but also be easy for users to understand and use.

As a system designer, I gained experience in organizing project ideas, reviewing documentation, and preparing the project overview and scope. This helped me think more from both the user's perspective and the developer's perspective.

### 🤝 Teamwork & Client Experience

This project also taught me the importance of teamwork and communication. Since each team member had different responsibilities, we had to discuss and coordinate our work properly to make sure the system design, documentation, and implementation were consistent.

Working on a client-based project also helped me understand that real systems must be designed based on actual user needs, not only based on what developers think is useful.

### 🔧 Suggested Improvements

The system could be improved by adding more automation features, such as **email or SMS notifications** when an application status changes. The report module could also be improved by adding **dashboard visualizations** to help admins and board directors understand application and financial records more easily.

Besides that, stronger security features such as **role-based access control**, **audit logs**, and **data encryption** could be added to better protect sensitive member information.

### 🌟 Overall Reflection

Overall, this project was meaningful because it showed me that software development is not only about coding. It also involves understanding user needs, planning carefully, designing proper workflows, communicating with clients, and continuously improving the system. This experience helped me build a stronger foundation in software engineering and system development.
